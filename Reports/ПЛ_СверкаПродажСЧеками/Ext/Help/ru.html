Сверка продаж с чеками (Портал)

Отчет предназначен для выявления расхождений между чеками общепита, загруженными из фронт-систем и проведёнными продажами.

Показатели отчета:

  • Выручка
  • Количество,
  • Сумма скидки,
  • Выручка без НДС,
  • Сумма НДС,

Каждый показатель делится на "Продажи", "Чеки", "Разница", где "Разница" = "Продажи" - "Чеки".

В показателях "Продажи" не учитываются передачи ТМЦ на "Шведскую линию", так как на них нет чеков.

Чеки учитываются все не помеченные на удаление, в том числе и те, по которым не закончено сопоставление и заполнение справочников.

В поле "Доп. аналитика" выводится значение, уточняющее продажу. Это может быть название скидки по Officer checks или отдельная группа финансового учёта, например для обособления санаторно-курортного лечения. 

Продажи по открытой цене группируются по номенклатуре открытой цены, для этого используется поле "Номенклатура чека" - в этом поле и из чеков и из продаж будет номенклатура открытой цены, в поле "Номенклатура" из чеков будет так же номенклатура открытой цены, из продаж - фактическая номенклатура продажи. Для остальных строк продаж номенклатура в полях "Номенклатура чека" и "Номенклатура" будет совпадать.

Поле "Вид отклонения" принимает следующие значения:

  • Возврат работ / услуг - если номенклатура является работой или услугой и количество по чекам за день по кассе ККМ меньше нуля.
  • Продажи и возвраты по разной стоимости - если по чекам по номенклатуре за день по кассе ККМ количество равно нулю, а сумма выручки при этом не нулевая.
  • В остальных случаях поле остаётся пустым.
