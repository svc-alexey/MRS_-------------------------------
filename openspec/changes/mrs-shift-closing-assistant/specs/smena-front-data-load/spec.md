# Delta for smena-front-data-load

## ADDED Requirements

### Requirement: Действие «(Пере)загрузить чеки за день» на шаге сверки
Шаг «Сверка чеков» ДОЛЖЕН предоставлять пользователю точечное действие «(Пере)загрузить чеки за день», инициирующее существующий механизм загрузки данных продаж фронт-систем (`ОбщийМодуль.MRS_МенеджерОбменаTNG.ВыполнитьЗагрузкуДанныхTNG`, `ОбщийМодуль.MRS_МенеджерОбменаMicros.ВыполнитьЗагрузкуДанныхMicros`) за выбранный в мастере день, без изменения алгоритма самой загрузки. Действие доступно независимо от того, есть ли на момент открытия шага уже загруженные за день данные — оно покрывает и случай полного отсутствия загрузки (автозагрузка ещё не выполнялась или завершилась ошибкой), и случай осознанной повторной загрузки при отклонениях сверки.

#### Scenario: Повторная загрузка при отклонениях, объяснимых более поздним закрытием чека
- GIVEN на шаге «Сверка чеков» за 2026-07-05 показано отклонение по чеку, которое пользователь предполагает объяснимым тем, что чек закрылся на фронте позже автозагрузки
- WHEN пользователь запускает действие «(Пере)загрузить чеки за день» за 2026-07-05
- THEN мастер повторно вызывает загрузку данных за 2026-07-05 для соответствующей фронт-системы через существующие общие модули без изменения их внутренней логики
- AND после завершения загрузки шаг «Сверка чеков» пересчитывает отклонения по актуальным данным

#### Scenario: Данных за день нет вообще
- GIVEN автозагрузка чеков за 2026-07-05 не выполнялась либо завершилась ошибкой, документов `Документ.питДанныеПродажФронта` за этот день не существует
- WHEN пользователь открывает шаг «Сверка чеков» за 2026-07-05
- THEN мастер показывает, что данных за день нет, и предлагает выполнить действие «(Пере)загрузить чеки за день» вместо показа пустой или ошибочной сверки

### Requirement: То же действие доступно на шаге сопоставления объектов фронта
Шаг «Сопоставление объектов фронта» ДОЛЖЕН также предоставлять точечное действие «(Пере)загрузить чеки за день», чтобы после того как пользователь сопоставил ранее несопоставленный объект фронта, соответствующие строки чеков были повторно обработаны существующим механизмом загрузки и учтены в загруженных данных.

#### Scenario: Перезагрузка после донастройки сопоставления номенклатуры
- GIVEN пользователь на шаге «Сопоставление объектов фронта» сопоставил номенклатуру, которая ранее была несопоставленной в чеках за 2026-07-05
- WHEN пользователь запускает действие «(Пере)загрузить чеки за день» на этом же шаге
- THEN мастер повторно вызывает существующий механизм загрузки за 2026-07-05
- AND строки чеков с этой номенклатурой обрабатываются на общих основаниях при следующем открытии шага «Сверка чеков»

### Requirement: Данные дня считаются доступными для сверки
Шаг «Сверка чеков» ДОЛЖЕН считать данные фронт-системы за выбранный день доступными для сравнения, когда за этот день загружен хотя бы один документ `Документ.питДанныеПродажФронта`. Как правило, эта доступность достигается автоматически — фоновыми заданиями `MRS_ЗагрузкаЧековTNG`/`MRS_ЗагрузкаЧековMicros`, выполняющимися до открытия мастера пользователем, без его участия — и не требует запуска действия «(Пере)загрузить чеки за день».

#### Scenario: Данные уже загружены фоновым заданием
- GIVEN фоновое задание `MRS_ЗагрузкаЧековMicros` загрузило чеки за 2026-07-05 автоматически до открытия мастера
- WHEN пользователь открывает шаг «Сверка чеков» за 2026-07-05
- THEN мастер считает данные дня доступными и выполняет сравнение без необходимости вручную запускать загрузку

### Requirement: Незакрытые на фронте чеки не считаются отдельной категорией отклонения
Чеки, не закрытые на фронт-системе на момент выполнения загрузки (автоматической или через действие «(Пере)загрузить чеки за день»), ДОЛЖНЫ отсутствовать в результате загрузки (отфильтровываются существующим механизмом загрузки) и НЕ ДОЛЖНЫ отображаться шагом «Сверка чеков» как отдельная категория отклонения или ошибки.

#### Scenario: Чек закроется позже
- GIVEN на кассе Micros есть чек, не закрытый на момент выполнения загрузки за 2026-07-05
- WHEN пользователь открывает шаг «Сверка чеков» за 2026-07-05
- THEN этот чек отсутствует в загруженных за 2026-07-05 данных
- AND шаг «Сверка чеков» не показывает по этому чеку отдельного предупреждения или ошибки
- AND после того как чек закроется на фронте и пользователь запустит действие «(Пере)загрузить чеки за день» за 2026-07-05, чек будет загружен и сверен на общих основаниях
