# Delta for smena-shift-closing-workflow

## ADDED Requirements

### Requirement: Выбор дня и складов на сессию мастера
Мастер ЗАКРЫТИЯ смены ДОЛЖЕН предлагать пользователю выбрать ровно один день (календарную дату) и один либо несколько складов (`Справочник.Склады`) из всех складов, зарегистрированных в информационной базе, перед началом работы с шагами.

#### Scenario: Открытие мастера без предварительного выбора
- GIVEN пользователь открывает мастер закрытия смены впервые за сессию
- WHEN форма выбора параметров отображается
- THEN пользователю доступен выбор одной даты и множественный выбор складов из полного списка `Справочник.Склады`
- AND без выбранной даты и хотя бы одного склада переход к шагам недоступен

#### Scenario: Несколько складов на одну сессию
- GIVEN пользователь выбрал день и три склада
- WHEN пользователь подтверждает выбор
- THEN мастер переходит к шагам, работая с этой комбинацией «день + три склада» как с единицей текущей сессии

### Requirement: Подсветка рекомендуемого дня начала
Мастер ДОЛЖЕН вычислять и подсвечивать пользователю рекомендуемый день начала работы — самый ранний календарный день среди дней, для которых хотя бы один шаг мастера не пройден хотя бы по одному из складов, доступных для выбора.

#### Scenario: Есть непройденные дни в прошлом
- GIVEN по складу «Склад-1» день 2026-07-05 не пройден (есть отклонения сверки), день 2026-07-06 не пройден (не сформированы целевые документы), день 2026-07-07 не начат
- WHEN пользователь открывает форму выбора параметров мастера
- THEN мастер подсвечивает 2026-07-05 как рекомендуемый день начала (самый ранний непройденный)

#### Scenario: Все дни по всем складам закрыты
- GIVEN для всех складов все дни до текущей даты запрета редактирования пройдены по всем шагам
- WHEN пользователь открывает форму выбора параметров мастера
- THEN мастер не подсвечивает конкретный рекомендуемый день и явно показывает, что непройденных дней не найдено

### Requirement: Жёсткая последовательность шагов
Мастер ДОЛЖЕН обеспечивать фиксированный порядок шагов — Сопоставление объектов фронта → Сверка чеков → Формирование целевых документов → Анализ отрицательных остатков — и НЕ ДОЛЖЕН допускать переход к работе на шаге N+1, пока шаг N не пройден для текущей комбинации день+склады. Загрузка чеков за день выполняется автоматически фоновым заданием до открытия мастера (см. domain `smena-front-data-load`) и не является отдельным шагом этой последовательности.

#### Scenario: Попытка перейти к сверке при незавершённом сопоставлении
- GIVEN для текущего дня есть хотя бы один несопоставленный объект фронта (касса или номенклатура), встреченный в загруженных чеках
- WHEN пользователь пытается открыть шаг «Сверка чеков» для работы
- THEN мастер не позволяет открыть шаг для работы и показывает, что предыдущий шаг «Сопоставление объектов фронта» не пройден

#### Scenario: Переход разрешён после завершения предыдущего шага
- GIVEN шаг «Сопоставление объектов фронта» пройден для текущего дня (все объекты фронта, встреченные в чеках дня, сопоставлены)
- WHEN пользователь открывает шаг «Сверка чеков»
- THEN шаг доступен для работы

#### Scenario: Возврат к предыдущему шагу вручную
- GIVEN пользователь находится на шаге «Сверка чеков» (уже открытом ранее)
- WHEN пользователь выбирает вернуться на шаг «Сопоставление объектов фронта» (например, чтобы досопоставить номенклатуру)
- THEN мастер разрешает открыть предыдущий уже пройденный шаг для повторной работы
- AND статусы шагов после него пересчитываются по актуальным данным при следующем открытии

### Requirement: Отсутствие автоматического отката статуса шага
Мастер НЕ ДОЛЖЕН автоматически откатывать статус пройденных шагов при изменении исходных данных (домэтчинг несопоставленного объекта фронта после прохождения шага сопоставления, (пере)загрузка чеков или переформирование выпусков после прохождения шагов «Сверка чеков»/«Формирование целевых документов»). Статус каждого шага пересчитывается по актуальным данным только при явном открытии этого шага пользователем.

#### Scenario: Домэтчинг номенклатуры после прохождения шага сопоставления
- GIVEN шаг «Сопоставление объектов фронта» пройден, пользователь перешёл к шагу «Сверка чеков»
- WHEN пользователь позже возвращается на шаг «Сопоставление объектов фронта» и сопоставляет ещё один объект, который раньше не встречался в чеках дня
- THEN мастер не откатывает автоматически статусы шагов «Сверка чеков»/«Формирование целевых документов», ранее показанные как пройденные
- AND пользователь должен вручную открыть нужный шаг, чтобы статус пересчитался по актуальным данным

### Requirement: Индикация готовности к продвижению даты запрета редактирования
Когда все шаги мастера пройдены для всех складов, выбранных в текущей сессии, для выбранного дня, мастер ДОЛЖЕН показать пользователю индикацию готовности («день чист по выбранным складам, дату запрета редактирования можно продвигать»). Мастер НЕ ДОЛЖЕН записывать значение в `РегистрСведений.ДатыЗапретаИзменения` — продвижение даты выполняется пользователем типовыми средствами ERP вне мастера.

#### Scenario: Все шаги пройдены по всем выбранным складам
- GIVEN в текущей сессии выбраны склады «Склад-1» и «Склад-2» на день 2026-07-05
- AND все четыре шага мастера пройдены для обоих складов на этот день
- WHEN пользователь просматривает итоговый статус сессии
- THEN мастер показывает индикацию готовности к продвижению даты запрета редактирования
- AND мастер не выполняет запись в `РегистрСведений.ДатыЗапретаИзменения`

#### Scenario: Не все шаги пройдены по одному из складов
- GIVEN в текущей сессии выбраны склады «Склад-1» и «Склад-2» на день 2026-07-05
- AND по «Склад-1» все шаги пройдены, по «Склад-2» шаг «Анализ отрицательных остатков» не пройден
- WHEN пользователь просматривает итоговый статус сессии
- THEN мастер не показывает индикацию готовности к продвижению даты
- AND явно указывает, что «Склад-2» блокирует готовность

### Requirement: Замена текущей плоской обработки закрытия смены
Мастер ДОЛЖЕН предоставлять весь функционал, ранее доступный через `DataProcessors/MRS_АРМ_Закрытия_смены` (запуск/повторный запуск загрузки данных, сверка, формирование ведомости остатков, анализ отрицательных остатков), в виде шагов и точечных действий текущей спецификации. `DataProcessors/MRS_АРМ_Закрытия_смены` НЕ ДОЛЖЕН дорабатываться в рамках этого change.

#### Scenario: Пользователь не обращается к старой обработке
- GIVEN мастер закрытия смены развёрнут и доступен пользователям
- WHEN ответственный выполняет закрытие дня
- THEN весь процесс выполняется через мастер без необходимости открывать `DataProcessors/MRS_АРМ_Закрытия_смены`
