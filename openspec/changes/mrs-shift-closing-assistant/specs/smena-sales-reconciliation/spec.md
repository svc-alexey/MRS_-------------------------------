# Delta for smena-sales-reconciliation

## ADDED Requirements

### Requirement: Построчный вывод отклонений сверки
Шаг «Сверка чеков» ДОЛЖЕН показывать каждое обнаруженное отклонение между данными фронт-системы и загруженными в 1С документами `Документ.питДанныеПродажФронта` как отдельную строку с указанием: чек/точка продаж, сумма по фронту, сумма по 1С, скидка по фронту (при наличии), скидка по 1С (при наличии). Шаг НЕ ДОЛЖЕН ограничиваться выводом только агрегированного показателя расхождения.

#### Scenario: Расхождение по сумме одного чека
- GIVEN по чеку №145 точки продаж «Бар-1» сумма по фронту 1 250,00 ₽, сумма по документу в 1С 1 200,00 ₽
- WHEN пользователь открывает шаг «Сверка чеков» за день, содержащий этот чек
- THEN мастер показывает отдельную строку по чеку №145 с суммой по фронту 1 250,00 ₽ и суммой по 1С 1 200,00 ₽

### Requirement: Явное «всё чисто» при отсутствии отклонений
Если по выбранному дню не найдено ни одного отклонения между данными фронта и загруженными документами, шаг «Сверка чеков» ДОЛЖЕН явно показать пользователю сообщение об отсутствии отклонений, а НЕ пустую таблицу без пояснения.

#### Scenario: День сверен без отклонений
- GIVEN за 2026-07-05 все чеки загружены и суммы/скидки по всем чекам совпадают между фронтом и 1С
- WHEN пользователь открывает шаг «Сверка чеков» за 2026-07-05
- THEN мастер показывает явное сообщение «отклонений не найдено» вместо таблицы

### Requirement: Любое ненулевое отклонение блокирует переход
Шаг «Сверка чеков» СЧИТАЕТСЯ пройденным только при полном отсутствии отклонений между данными фронта и 1С за выбранный день. Порогов допустимого расхождения (по сумме или по проценту) НЕ ДОЛЖНО существовать — любое ненулевое отклонение блокирует переход к следующему шагу.

#### Scenario: Отклонение на 1 копейку
- GIVEN по одному чеку расхождение между фронтом и 1С составляет 0,01 ₽
- WHEN мастер вычисляет статус шага «Сверка чеков»
- THEN шаг отмечается как не пройденный
- AND переход к шагу «Формирование целевых документов» для этого дня недоступен

### Requirement: Отклонения по алкоголю и маркам видны как отдельная категория
Шаг «Сверка чеков» ДОЛЖЕН явно выделять строки отклонений, связанные с недостатком данных по акцизным маркам в фискальной базе Micros (когда сумма/состав чека с алкоголем не сходится из-за отсутствующей марки), отдельно от прочих отклонений по сумме, чтобы пользователь понимал природу проблемы без обращения к фискальному регистратору напрямую.

#### Scenario: Чек с алкоголем без марки в фискальной базе
- GIVEN чек Micros с продажей алкоголя порциями содержит расхождение, вызванное отсутствием марки в фискальной базе на момент продажи
- WHEN пользователь открывает шаг «Сверка чеков»
- THEN строка отклонения по этому чеку помечена как связанная с отсутствием марки, а не как обычное расхождение по сумме
