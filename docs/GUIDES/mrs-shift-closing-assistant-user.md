# Помощник закрытия смены

Документ для пользователей.

## Зачем нужен помощник

Помощник закрытия смены нужен, чтобы перед закрытием дня быстро проверить: продажи с касс загружены, объекты фронта сопоставлены с объектами 1С, выпуск блюд сформирован, а по складам нет отрицательных остатков.

Простыми словами: это контрольный список перед тем, как считать день закрытым. Он не заменяет учетные документы, а помогает найти проблемы раньше, пока их еще можно исправить.

Помощник подготавливает смену к закрытию. По текущему коду он не является отдельным документом финального закрытия периода и не подтвержден как механизм переноса даты запрета изменения.

## Где открыть

В расширении есть обработка `MRS_ПомощникЗакрытияСмены`.

В исходниках она лежит здесь:

- `DataProcessors/MRS_ПомощникЗакрытияСмены.xml`
- `DataProcessors/MRS_ПомощникЗакрытияСмены/Forms/Форма/Ext/Form.xml`
- `DataProcessors/MRS_ПомощникЗакрытияСмены/Forms/Форма/Ext/Form/Module.bsl`

В интерфейсе синоним обработки: **Помощник закрытия смены (mrs)**.

## Как начать работу

1. Откройте помощник.
2. Выберите день, который нужно проверить.
3. Отметьте точки продаж.
4. Нажмите **Начать работу**.

Если помощник пишет, что день не выбран, не отмечена точка продаж или для точки продаж не заполнен склад, дальше идти рано. Сначала исправьте это.

Кнопка **Получить рекомендуемый день отработки** подсказывает самый ранний день, где по выбранным точкам продаж еще есть незакрытые проблемы. Подсказка смотрит данные 1С от даты запрета изменения до вчерашнего дня.

## Откуда берутся точки продаж

Список точек продаж строится по кассам ККМ.

Точка появится в списке, если одновременно выполнено:

- касса ККМ не помечена на удаление;
- касса сопоставлена с фронт-системой в регистре `ПЛ_СоответствиеСФронтСистемами`;
- у кассового узла заполнен склад приготовления;
- фронт-система относится к TNG или Micros.

Если нужной точки нет в списке, обычно надо проверить сопоставление кассы и настройку кассового узла.

## Шаг 1. Сопоставление объектов фронта

На этом шаге помощник проверяет, понимает ли 1С, что пришло из кассы.

Он ищет:

- кассы фронта, которые не связаны с кассой ККМ в 1С;
- кнопки или блюда фронта, которые не связаны с номенклатурой в 1С;
- строки чеков, где номенклатура в 1С пустая.

Если в таблице есть строки, выберите правильный объект 1С и нажмите **Сопоставить**. После этого помощник обновит ссылки в уже загруженных чеках.

Данные берутся из документов `питДанныеПродажФронта` и регистра `ПЛ_СоответствиеСФронтСистемами`.

## Шаг 2. Сверка чеков

На этом шаге помощник сравнивает продажи фронта с тем, что уже загружено в 1С.

Он проверяет:

- есть ли загруженные чеки за выбранный день и точки продаж;
- совпадают ли суммы по строкам;
- совпадают ли скидки;
- есть ли отдельные проблемы по алкоголю и маркам.

Если данных нет или есть расхождения, используйте **(Пере)загрузить чеки за день**. Помощник запустит загрузку из TNG и Micros в фоне, а после завершения надо перепроверить шаг.

Данные фронта запрашиваются через `MRS_МенеджерОбменаTNG` и `MRS_МенеджерОбменаMicros`. Загруженные чеки лежат в `Документ.питДанныеПродажФронта`.

## Шаг 3. Формирование целевых документов

На этом шаге помощник проверяет выпуски блюд.

Сначала он убеждается, что все чеки фронта есть в 1С. Если загрузка неполная, выпуск формировать нельзя: сначала перезагрузите чеки.

Если помощник видит, что выпуски устарели, он задает вопрос:

> Рекомендуется пересоздание выпусков блюд для обновления. Выполнить сейчас?

Если ответить **Да**, помощник запустит формирование целевых документов. Основной результат здесь — документы `питВыпускБлюд`.

Помощник также показывает:

- расхождения между количеством по чекам и количеством в выпусках;
- непроведенные выпуски;
- документы по открытым ценам, которые требуют внимания.

Данные берутся из `питДанныеПродажФронта`, `питВыпускБлюд`, `питУчетДанныхПродаж` и существующей обработки `питЗагрузкаИзДанныхПродажФронта`.

## Шаг 4. Отрицательные остатки

На этом шаге помощник проверяет, не ушел ли склад в минус после движений дня.

Если минусы есть, сформируйте ведомость отрицательных остатков и разберите строки. Ведомость показывает склад, номенклатуру, характеристику, конечный остаток и документ-регистратор, который связан с расходом дня.

Данные берутся из регистра накопления `ТоварыНаСкладах`.

## Шаг 5. Итоговый отчет

Итоговый отчет собирает короткую сводку:

- сколько осталось несопоставленных объектов фронта;
- сколько отклонений найдено при сверке чеков;
- сколько отклонений, ошибок и документов открытых цен есть по выпускам;
- сколько отрицательных остатков найдено.

Если в отчете по всем шагам все чисто, день можно считать подготовленным к закрытию.

## Что делать при типовых проблемах

| Проблема | Что означает | Что сделать |
|---|---|---|
| Нет точки продаж в списке | Касса не попала в отбор помощника | Проверить кассу ККМ, сопоставление кассы с фронтом и склад приготовления в кассовом узле |
| Есть несопоставленные объекты | Код фронта не связан с объектом 1С | Заполнить объект 1С в строке и нажать **Сопоставить** |
| Нет данных для сверки | За день не найдено загруженных чеков | Нажать **(Пере)загрузить чеки за день** |
| Суммы или скидки не совпадают | Фронт и 1С видят разные данные по чеку | Перезагрузить чеки, затем перепроверить шаг |
| Загрузка чеков неполная | Не все чеки фронта присутствуют в 1С | Перезагрузить чеки и снова проверить формирование |
| Выпуски устарели | Чеки изменились, а документы выпуска еще не обновлены | Согласиться на пересоздание выпусков |
| Документ выпуска не проведен | При формировании была ошибка | Открыть документ/ошибку и исправить причину |
| Есть отрицательные остатки | На складе товара стало меньше нуля | Открыть ведомость отрицательных остатков и исправить учетную причину |

## Важное ограничение

Помощник показывает состояние и запускает исправляющие действия, но не отменяет необходимость проверить результат глазами. Если таблица шага заполнена проблемами, не считайте шаг закрытым только потому, что удалось перейти дальше по страницам.
